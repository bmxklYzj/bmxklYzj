Number类型

  1. 进制
    十进制：最基本的进制格式
    八进制：八进制字面值的第一位必须是零（0），然后是八进制数字序列（0～7）。如果字面值中的数值超出了范围，那么前导零将被忽略，后面的数值将被当作十进制数值解析。
    十六进制：十六进制字面值的前两位必须是 0x，后跟任何十六进制数字（0～9 及 A～F）。
  2. Infinite
    Infinite:如果超出了 Number.MAX_VALUE 则数值成为 Infinite，对应的有Number.MIN_VALUE、-Infinite
    isFinite()函数可以用来判断数字是否是有穷的，位于-Infinite和Infinite之间

  1. 数值转换
    Number()/parseInt()/parseFloat()


对比Number()/parseInt()观察可以看到，在转换null和空字符串时我们期待的结果应该是NaN，parseInt()转换是正确的，然而Number()则很诡异，所以转换整数的时候不要用Number()函数，用parseInt()就够了

使用parseInt()函数的时候要加上第二个参数表示转换的基数

parseInt()会忽略字符串前面的空格，直至找到第一个非空格字符。如果第一个字符不是数字字符或者负号，parseInt()就会返回 NaN；也就是说，用 parseInt()转换空字符串会返回 NaN（Number()对空字符返回 0）。如果第一个字符是数字字符，parseInt()会继续解析第二个字符，直到解析完所有后续字符或者遇到了一个非数字字符。

String类型

把一个值转换为字符串：

  1. toString(),此函数可以传递第二个参数表示：输出数值的基数

    数值、布尔值、对象和字符串值（没错，每个字符串也都有一个 toString()方法，该方法返回字符串的一个副本）都有 toString()方法。但 null 和 undefined 值没有这个方法。

  2. String(),转换任意值为字符串

如果值有 toString()方法，则调用该方法（没有参数）并返回相应的结果；
 如果值是 null，则返回”null”；
 如果值是 undefined，则返回”undefined”。

注意两个函数用法的区别： value.toString([可选的基数]) / String(value)

Object类型

Object 类型是所有它的实例的基础。Object 类型所具有的任何属性和方法也同样存在于更具体的对象中。
Object的每个实例都有7个方法:
constructor() 构造对象的函数
hasOwnProperty(propertyName) 属性是否存在于当前实例中（而不是在实例的原型中）
isPropertyOf(name) 当前对象是否是传入对象的原型
propertyIsEnumable() 属性是否可枚举
toLocalString() 返回与地区对于的字符串表示
toString()
valueOf()

由于在 ECMAScript 中 Object 是所有对象的基础，因此所有对象都具有这些基本的属性和方法。

js中获取宽度

首先要明确我们所说的宽度有两种

charpter 2 在 HTML 中使用 JavaScript

script 标签

  1. async：可选。表示应该立即下载脚本，但不应妨碍页面中的其他操作，比如下载其他资源或等待加载其他脚本。只对外部脚本文件有效。
    async 只适用于外部脚本文件，并告诉浏览器立即下载文件。但与 defer不同的是，标记为 async 的脚本并不保证按照指定它们的先后顺序执行。
    指定 async 属性的目的是不让页面等待两个脚本下载和执行，从而异步加载页面其他内容。为此，建议异步脚本不要在加载期间修改 DOM。

  2. defer：可选。表示脚本可以延迟到文档完全被解析和显示之后再执行。只对外部脚本文件有效。IE7 及更早版本对嵌入脚本也支持这个属性。HTML5 规范要求脚本按照它们出现的先后顺序执行，各浏览器实现的时候不一定按顺序执行。
    脚本会被延迟到整个页面都解析完毕后再运行。相当于告诉浏览器立即下载，但延迟执行。

无论如何包含代码，只要不存在 defer 和 async 属性，浏览器都会按照<script>元素在页面中出现的先后顺序对它们依次进行解析。换句话说，在第一个<script>元素包含的代码解析完成后，第二个<script>包含的代码才会被解析，然后才是第三个、第四个……

不要把script标签放在头部，应该放在</body>前面，因为script标签一旦出现便会阻塞页面的渲染，导致页面出现白屏。而放在底部则不会阻塞页面的渲染。用户也会因为浏览器窗口显示空白页面的时间缩短而感到打开页面的速度加快了。

在包含外部 JavaScript 文件时，必须将 src 属性设置为指向相应文件的 URL。而这个文件既可以是与包含它的页面位于同一个服务器上的文件，也可以是其他任何域中的文件。和image类似，可以跨域

 使用 defer 属性可以让脚本在文档完全呈现之后再执行。延迟脚本总是按照指定它们的顺序执行。
 使用 async 属性可以表示当前脚本不必等待其他脚本，也不必阻塞文档呈现。不能保证异步脚本按照它们在页面中出现的顺序执行。

我们在访问对象的属性的时候有两种方式，.号和[]号，左边的是 base value，右边的是 reference name。我称之为基值和引用值，当左边的基值不存在的时候就会出现 ReferenceError，

TypeError 是类型转换错误，尝试着把 unll 和 undefined 当成对象来访问就会出现转型错误

关于执行环境、变量对象、活动对象参考：http://programmers.stackexchange.com/questions/189967/what-is-an-activation-object-in-javascript

执行环境：

变量前是否用var

我们都知道js中这样一句（非严格模式下）
a = 1 ，那么 a 会成为window的属性，没有问题。
但是直接访问没有定义的 a 会出现 ReferenceError，此时a并不会成为全局window的属性，也就是说其左边的基值没有定义，所以出现 ReferenceError，这就比较矛盾了（但在es5严格模式下，二者表现相同，都会报错）, 编译器在看到a赋了值时会把a追加成window的属性，但没有给a赋值时不会把a追加成window的属性。

短路运算符 && 和 ||

会转化为false的表达式有: 0, “”, unll, undefined

exper1 && exper2 如果expr1 能转换成false则返回expr1,否则返回expr2（注意返回的是表达式的值，而不仅仅是true或false）

exper1 && exper2 如果expr1 能转换成true则返回expr1,否则返回expr2

利用短路操作符可以简化代码，如我们经常会遇到访问undefined的属性，这样的话会报typeError的错误，解决：http://harttle.com/2016/01/30/javascript-objects.html

undefined和null在if语句中，都会被自动转为false，相等运算符甚至直接报告两者相等，但不全等
根据C语言的传统，null被设计成可以自动转为0。
null是一个表示”无”的对象，转为数值时为0；undefined是一个表示”无”的原始值，转为数值时为NaN
>
首先，null像在Java里一样，被当成一个对象。但是，JavaScript的数据类型分成原始类型（primitive）和合成类型（complex）两大类，Brendan Eich觉得表示”无”的值最好不是对象。
其次，JavaScript的最初版本没有包括错误处理机制，发生数据类型不匹配时，往往是自动转换类型或者默默地失败。Brendan Eich觉得，如果null自动转为0，很不容易发现错误。
因此，Brendan Eich又设计了一个undefined。

>
null和undefined基本是同义的，只有一些细微的差别。
null表示”没有对象”，即该处不应该有值。典型用法是：
（1） 作为函数的参数，表示该函数的参数不是对象。
（2） 作为对象原型链的终点。

Object.getPrototypeOf(Object.prototype)
// null
undefined表示”缺少值”，就是此处应该有一个值，但是还没有定义。典型用法是：
（1）变量被声明了，但没有赋值时，就等于undefined。
（2) 调用函数时，应该提供的参数没有提供，该参数等于undefined。
（3）对象没有赋值的属性，该属性的值为undefined。
（4）函数没有返回值时，默认返回undefined。

var i;
i // undefined
function f(x){console.log(x)}
f() // undefined
var o = new Object();
o.p // undefined
var x = f();
x // undefined

来自 http://www.ruanyifeng.com/blog/2014/03/undefined-vs-null.html

##
null 表示一个值被定义了，定义为“空值”；
undefined 表示根本不存在定义。
所以设置一个值为 null 是合理的，如
objA.valueA = null;
但设置一个值为 undefined 是不合理的

gulp 使用

gulp的启动文件是gulpfile.js，gulp只有四个函数

src(globs[, options])

globs是匹配模式，可以是目录、文件的路径或者是数组
options是读取文件是的参数，一般不用

dest(path[, options])

输出目录不存在则会自动创建，第二个参数cwd可以指定输出目录的基目录（默认是当前目录），mode可以指定文件权限

task(name, [,deps], fn)

name：任务名
deps数组：deps数组中的任务会先于当前任务执行
fn：该函数定义任务所要执行的一些操作。通常来说，它会是这种形式：gulp.src().pipe(someplugin())

如果一个任务的名字为default，就表明它是“默认任务”，在命令行直接输入gulp命令，就会运行该任务。

gulp.watch(glob[, opts], tasks)

watch方法用于指定需要监视的文件。一旦这些文件发生变动，就运行指定任务。
另一种写法是watch方法所监控的文件发生变化时（修改、增加、删除文件），会触发change事件。可以对change事件指定回调函数。
