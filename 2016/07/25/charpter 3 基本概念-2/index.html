charpter 3 基本概念

语法

变量命名规则：

第一个字符必须是一个字母、下划线（_）或一个美元符号（$）；
其他字符可以是字母、下划线、美元符号或数字。

对比c++
第一个字符必须是字母、下划线
其余字符必须是数字、字母、下划线
js中只是多了个$而已

js严格模式

所谓严格模式就是为js定义一种不同的解释与执行模型，使得es3中的一些不确定的行为得到得到处理，对一些不安全的操作也会抛出错误

这行代码看起来像是字符串，而且也没有赋值给任何变量，不会破坏 es3 的语法，但其实它是一个编译指示（pragma），用于告诉支持的 JavaScript 引擎切换到严格模式。

js语句结束的分号

虽然语句结尾的分号不是必需的，但我们建议任何时候都不要省略它。

js变量

ECMAScript 的变量是松散类型的，所谓松散类型就是可以用来保存任何类型的数据。换句话说，每个变量仅仅是一个用于保存值的占位符而已。

1种复杂类型：Object(本质上是由一组无序的名值对组成的)

Boolean类型

只有两个取值： true/false

Number类型

  1. 进制
    十进制：最基本的进制格式
    八进制：八进制字面值的第一位必须是零（0），然后是八进制数字序列（0～7）。如果字面值中的数值超出了范围，那么前导零将被忽略，后面的数值将被当作十进制数值解析。
    十六进制：十六进制字面值的前两位必须是 0x，后跟任何十六进制数字（0～9 及 A～F）。
  2. Infinite
    Infinite:如果超出了 Number.MAX_VALUE 则数值成为 Infinite，对应的有Number.MIN_VALUE、-Infinite
    isFinite()函数可以用来判断数字是否是有穷的，位于-Infinite和Infinite之间
  3. NaN
    not a number 不是一个数字，

注意：0/0 返回NaN
任何有NaN参与的运算返回的都是NaN

0/非零 返回0
0/0 或 0/NaN 返回NaN

isNaN()函数 用来判断是否是数字

es6 中新增了判断是否是整数的函数 isInteger()

  1. 数值转换
    Number()/parseInt()/parseFloat()


对比Number()/parseInt()观察可以看到，在转换null和空字符串时我们期待的结果应该是NaN，parseInt()转换是正确的，然而Number()则很诡异，所以转换整数的时候不要用Number()函数，用parseInt()就够了

使用parseInt()函数的时候要加上第二个参数表示转换的基数

parseInt()会忽略字符串前面的空格，直至找到第一个非空格字符。如果第一个字符不是数字字符或者负号，parseInt()就会返回 NaN；也就是说，用 parseInt()转换空字符串会返回 NaN（Number()对空字符返回 0）。如果第一个字符是数字字符，parseInt()会继续解析第二个字符，直到解析完所有后续字符或者遇到了一个非数字字符。

String类型

把一个值转换为字符串：

  1. toString(),此函数可以传递第二个参数表示：输出数值的基数

    数值、布尔值、对象和字符串值（没错，每个字符串也都有一个 toString()方法，该方法返回字符串的一个副本）都有 toString()方法。但 null 和 undefined 值没有这个方法。

  2. String(),转换任意值为字符串

如果值有 toString()方法，则调用该方法（没有参数）并返回相应的结果；
 如果值是 null，则返回”null”；
 如果值是 undefined，则返回”undefined”。

注意两个函数用法的区别： value.toString([可选的基数]) / String(value)

Object类型

Object 类型是所有它的实例的基础。Object 类型所具有的任何属性和方法也同样存在于更具体的对象中。
Object的每个实例都有7个方法:
constructor() 构造对象的函数
hasOwnProperty(propertyName) 属性是否存在于当前实例中（而不是在实例的原型中）
isPropertyOf(name) 当前对象是否是传入对象的原型
propertyIsEnumable() 属性是否可枚举
toLocalString() 返回与地区对于的字符串表示
toString()
valueOf()

由于在 ECMAScript 中 Object 是所有对象的基础，因此所有对象都具有这些基本的属性和方法。
